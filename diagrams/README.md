# Diagrams Folder

This folder contains all **diagrams and visual representations** used in the project.  
These diagrams help explain the robot system, behaviour, and workflow for both the team and the markers.

---

## Typical Files

### 1. `fsm_diagram.png` / `.pdf`

- **Purpose:** Shows the robot’s Finite State Machine (FSM) or behaviour tree.
- Illustrates the **states**, **transitions**, and **conditions** for each behaviour.

### 2. `system_architecture.png` / `.pdf`

- **Purpose:** Shows the **Sense → Process → Decide → Act** flow.
- Illustrates how sensor data flows to the controller, triggers decision logic, and drives actuators.

### 3. Optional Diagrams

- Any other diagrams supporting the project, such as:
  - Environment layout schematic
  - Sensor placement diagram
  - Flowcharts for decision logic

---

## Notes

- Use clear and readable file names so team members and markers can easily identify diagrams.
- Keep images in standard formats (PNG, JPG, PDF) for easy viewing.
